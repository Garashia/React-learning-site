## Zenn/Qiitaの可読性とUI・Webデザイン原則の比較検証（チェックリスト付き）

目的: 学習用サイトの可読性・操作性を最大化するため、Zenn/Qiitaに共通する「見やすさの要素」と一般UI/Webデザインの原則を照合し、実装に落とし込めるチェックリストを作成する。

---

### 1. ページ構造と視線誘導（Zenn/Qiita観察）
- 上部固定ナビ（軽量・明確な階層）
- 本文の左右パディングは広め、行長は約60–80字（英40–75cpl）
- セクション見出しと本文の明確な階層（H2>H3>本文）
- コードブロックは余白多め＋等幅フォント＋シンタックスハイライト＋コピー
- 余白と罫線は最小限。ブロック間は「余白＞線」で区切る

チェックリスト
- [ ] ヘッダーは高さ控えめでスクロール固定／影で層分け
- [ ] 本文コンテナは max-width 720–840px（IDE連動時は段組でも行長を守る）
- [ ] H2は本文より1.5–1.8倍、H3は1.2–1.4倍、余白は上>下
- [ ] 段落間マージンは行高の1.0–1.5倍、段落頭は字下げしない
- [ ] コードは`pre`内パディング 12–16px、行高1.5、背景は低コントラストの面
- [ ] コードコピー/言語ラベル/行番号（任意）

---

### 2. タイポグラフィ（和文/欧文混在）
- 和文本文: 16–18px, line-height 1.7–1.9
- 英文/コード: 等幅は 13–15px, line-height 1.5–1.7
- 行長: 和文 45–70字、欧文 50–75cpl を目安
- 見出しは字間をわずかに詰め、本文は詰めない

チェックリスト
- [ ] ベースフォントサイズは 16px 以上、媒体に応じて流体タイポを採用
- [ ] 日本語UIは Noto Sans JP/Inter 等の可読ファミリ
- [ ] 数字/コードの等幅は UIの本文と別フォントで指定
- [ ] 句読点ぶら下がり/和文合字は無効（デザイン方針に応じて）

---

### 3. 色・コントラスト・意味付け
- 文字と背景は WCAG AA: 通常4.5:1以上、見出し/太字は3:1以上
- 情報色は状態に一貫（primary/success/warn/error/info）
- デフォルトは低飽和、ホバー/アクティブで彩度と明度を段階的に変化

チェックリスト
- [ ] 主要テキストと背景のコントラスト比を測定（AA/AAA）
- [ ] フィードバックの色を領域/テキスト/アイコンで冗長化（色みに依存しない）
- [ ] ダークモード時は彩度を落としコントラストを抑制

---

### 4. ナビゲーション/情報設計
- 上位ナビ＋ホバードロップダウン（遷移の探索コストを低減）
- 現在地の強調（アンダーライン/色/太字）
- パンくず/戻る導線/検索の常設

チェックリスト
- [ ] ナビ要素はタップ領域44px以上
- [ ] ドロップダウンはホバー/フォーカスの両方で開閉、外側クリックで閉じる
- [ ] 現在地/無効状態/外部リンクの表現が一貫

---

### 5. コードブロック体験（Qiita/Zenn準拠＋発展）
- コピー/言語タブ/行番号/差分表示/長行のソフトラップ
- 見出し直後に短い要点（何が分かるか）

チェックリスト
- [ ] `Copy` ボタン、成功トースト、キーボードコピー（Ctrl/Cmd+C）
- [ ] 長い行は`white-space: pre-wrap;`で折返し（横スクロールを避ける）
- [ ] 差分は+/-の色分けと背景帯
- [ ] 行頭の余白/番号のコントラストを下げ、本文より目立たせない

---

### 6. インタラクション/動作フィードバック
- 読了率/目次のスクロール同期（progress/scrollspy）
- ローディング: Skeleton or Progressive（ちらつき防止）
- フォーカスリングはカスタムだが必ず可視

チェックリスト
- [ ] 重要操作には押下/成功/失敗の3段階フィードバック
- [ ] 非同期待機>500msでローディング表示、>2sで詳細（処理内容/再試行）
- [ ] キーボード操作で全機能利用可能（Tab順/ESC閉じる）

---

### 7. アクセシビリティ（WCAG 2.2/JA）
- 見出し順序/ラベル/ランドマーク（header/main/nav）
- コントラスト/フォーカス可視/入力支援/エラーの特定

チェックリスト
- [ ] 画像・アイコンに代替テキスト、装飾は空alt
- [ ] Live領域（トースト/バリデーション）は`aria-live`で通知
- [ ] モーション削減`prefers-reduced-motion`対応
- [ ] 言語属性`lang="ja"`、タイトル/descriptionの有無

---

### 8. レスポンシブ/レイアウトグリッド
- モバイルファースト、段組は1→2→3の順で拡張
- コンテナの内側余白はブレークポイントごとに微増

チェックリスト
- [ ] ブレークポイント例: 480/768/1024/1280
- [ ] カード/リストは密度を切替（compact/regular）
- [ ] 画像は`aspect-ratio`でレイアウトシフトを抑制

---

### 9. パフォーマンス/可用性
- 1st input delay/CLS/LCPの監視、フォントの遅延読込
- 画像の`srcset`/`sizes`、コード分割、キャッシュ制御

チェックリスト
- [ ] LCP<2.5s, CLS<0.1, INP<200ms を目標
- [ ] Webフォントは`display=swap`、先頭描画はシステムフォント
- [ ] 重要UIのレイアウトはCSSで先出し（JS依存を最小）

---

### 10. 執筆UI/学習サイト特有の最適化
- IDEモード（ファイルツリー/高さ拡張）と読み物モードをトグル
- 解説は「スニペット→要点→落とし穴→演習」の粒度
- 目次/見出しリンク/段落番号で再訪問性を高める

チェックリスト
- [ ] スニペット直上に「学べること」3点以内
- [ ] 章末に演習の自動採点（DOM/テスト）
- [ ] コード/解説の対応をアンカーリンクで相互参照

---

### 付録: 実装スタイル指針（抜粋CSSトークン）

```css
  --container-max: 840px;
  --space-1: 4px; --space-2: 8px; --space-3: 12px; --space-4: 16px; --space-6: 24px;
  --text-body: 16px; --lh-body: 1.8; --text-code: 14px; --lh-code: 1.6;
  --fg: #111827; --fg-weak:#6b7280; --bg:#ffffff; --line:#e5e7eb;
}
main{max-width:var(--container-max);margin:0 auto;padding:var(--space-4);} 
article{line-height:var(--lh-body);font-size:var(--text-body);} 
pre code{font-size:var(--text-code);line-height:var(--lh-code);} 
```
